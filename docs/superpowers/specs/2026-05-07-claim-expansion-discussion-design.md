# Jenny 案訴之擴張討論頁設計

日期：2026-05-07

## 目的

新增一頁供原告與律師討論「訴之擴張」策略的 GitHub Pages 頁面，定位不是給法院直接閱讀的正式書狀，而是整理所有可能求償項目、法律基礎、現有證據與缺失候補，讓律師可快速判斷哪些項目要立即納入、哪些項目待補證後再擴張。

此頁的核心策略有二：

- 保本主軸要優先確保 `修繕金 + 鑑定費 + 木地板及附帶裝修損害 + 恢復原有居住品質所必要的直接支出` 足額回復，不讓 Jenny 自掏腰包承擔原本不應承擔的居住損失。
- 其他損害項目與非財產上損害不先刪除，而是全部列入討論範圍，讓律師依證據、程序與訴訟策略自行判斷是否納入訴之擴張。

## 頁面定位

- 頁面名稱：`claim-expansion.html`
- 類型：公開可讀的 GitHub Pages 策略討論頁
- 語氣：專業、直接、供律師討論，不寫成法官裁判書風格，也不使用情緒化語言
- 風險處理：不寫明內部談判策略或分成考量，但頁面內容本身要足以支持律師評估擴張空間

## 內容結構

### 1. 策略定位

頁首先用短段落說明：

- 本頁目的在於盤點所有可供律師評估之訴之擴張項目
- 保本主軸是回復原有居住品質與避免原告自行負擔必要修復成本
- 其餘項目則作為策略擴張範圍，供律師視證據、鑑定結果與程序位置判斷是否納入

### 2. 求償項目總表

以矩陣表為主，每列一個項目，欄位固定為：

- 分類
- 項目
- 法律基礎
- 可主張理由
- 目前已有證據
- 缺失候補
- 可量化方式
- 與律師討論重點

### 3. 證據缺失候補清單

把跨項目的證據缺口抽出成獨立段落，例如：

- 最新修繕估價單
- 木地板拆除 / 重鋪 / 發霉照片與估價
- 除霉、除蟲、清潔、暫時處置單據
- 遷居、住宿、搬運、倉儲單據
- 醫療紀錄
- 修復期間不可居住之鑑定承接

### 4. 與律師討論的決策點

頁尾列出數個明確問題，讓律師直接決定：

- 保本主軸目前應先擴張到哪裡
- 哪些項目需待鑑定結果出來後再定金額
- 哪些項目可先列原因事實、暫不寫死金額
- 精神慰撫金與健康損害是否保留在本次擴張或下一次補充

## 分類原則

### 保本主軸

以下項目歸入保本主軸：

- 漏水本體修復費
- 鑑定費與必要檢測費
- 木地板損害
- 附帶裝修復原
- 恢復原有居住品質所必要的直接支出

這一類的頁面論述基調是：

> 原告原本即應取得可正常、安全、完整使用之居住環境；凡屬回復該狀態所必要之費用，不應由原告自行負擔。

### 策略擴張項

以下項目全列入策略擴張討論範圍：

- 修復期間遷居費
- 住宿費
- 搬運費 / 倉儲費
- 除霉 / 除蟲 / 清潔費
- 居住空間縮減或部分失能損害
- 健康損害
- 精神慰撫金

此區不預先刪項，也不先下「不建議主打」結論，而是用證據現況與缺失候補提示律師判斷風險。

## 呈現原則

- 金額欄保留；未定者標 `待估 / 待單據 / 待鑑定`
- 頁面明確區分「已有單據可量化」與「仍需待鑑定或補件」
- 不把項目寫成已經全部成熟成立，而是如實標示目前證據成熟度
- 對於高風險項目，例如精神慰撫金，不刪除，但會明示證據門檻與程序風險

## 與現有頁面關係

- 這一頁不取代 `timeline.html`
- `timeline.html` 可在損害爭點或來源區新增入口
- 視需要也可在首頁 `index.html` 新增「訴之擴張討論版」入口，但不與正式證據卡混在一起

## 驗證標準

- 讀者一眼可分辨哪些屬保本主軸，哪些屬策略擴張
- 每個求償項目都能看到至少一個法律基礎與一個證據現況判斷
- 證據不足的地方已明確列為 `缺失候補`
- 頁面足以支撐與律師進行訴之擴張會議，而不需先翻原始對話逐項重整
