# Jenny 案時間軸頁補述設計

日期：2026-05-07

## 目的

在現有 `docs/timeline.html` 與 `docs/index.html` 之上，補入使用者於 `[LINE]請勿直接電話聯繫🍣♨️Jenny.txt` 第 `1290` 行後新增的對話內容，並維持頁面仍以「案件閱讀與整理」為主，而非直接改寫成書狀。

## 已確認方向

- 每個時間軸事件直接標示 `前手 / 後手 / 程序`
- 新增內容採「平衡版」
- 新內容中，`未驗收催尾款 / 修瑕疵綁付款 / 付款不等於驗收` 作為輔助點，用來評價後手履約誠信與抗辯可信度
- 此段不作為漏水主因證明主體，不與鑑定、施工照片、合約範圍、防水保固混寫

## 要補入的核心事實

### 後手履約與驗收脈絡

- `2023-03-31`：後手進場
- `2023-08-01`：原告因房租壓力先行搬入，但當時工程未完工、未正式驗收
- `2023-10` 前後：工程仍未完成，卻已催付尾款
- `2023-12-01`：原告於未正式驗收、瑕疵未排除的脈絡下支付尾款
- `2024-08`：原告詢問驗收程序，被告法代稱「工務還沒跟你驗收嗎」，反證驗收程序本應存在

### 與前手脈絡分流

- `2024-01-28` 發現壁癌 / 漏水，於頁面上標示為 `前手`
- `2024年8月尚未報出後手漏水` 的敘述，應標示為 `後手` 線，不與前手漏水發現時間混寫

## 頁面修改策略

### `docs/timeline.html`

- 保留既有主架構
- 在時間軸事件標題或事件籤上加入 `前手 / 後手 / 程序`
- 新增一段 `後手：入住、未驗收與尾款壓力`
- 新增一張爭點卡：
  - `入住及付款不等於驗收完成`
- 新增一個輔助區塊：
  - 標題偏中性，例如 `後手履約誠信爭議（輔助評價點）`
  - 明寫用途：
    - 不直接證明漏水原因
    - 但可作為評價後手履約誠信及抗辯可信度的輔助資料

### `docs/index.html`

- 視版面情況補一張新證據卡
- 主題為：
  - `未完工 / 未驗收爭議`
- 若加入圖片，使用使用者提供的 `Image 1703.png` 作為輔助圖，而非主證圖

## 頁面語氣原則

- 保持中性整理語氣
- 不使用 `半哄半騙`、`話術`、`欺負老實人` 等主觀詞
- 法律效果可以寫硬，例：
  - 原告搬入係因房租壓力，非正式驗收完成之表示
  - 原告付款時工程尚未正式驗收，瑕疵亦未完全排除
  - 付款背景不足以直接推論原告已承認工程完成、品質無瑕疵或放棄後續權利

## 不納入主頁主體的內容

- 對律師收費、服務感受、補付 `20,000` 等內容
- 對人格、態度、說話方式的直接評價

## 驗證標準

- `timeline.html` 可清楚看出 `前手 / 後手 / 程序` 三條脈絡
- 新增內容不會讓讀者誤解為「尾款爭議 = 漏水主因」
- 頁面仍維持閱讀版風格，而不是變成完整書狀
