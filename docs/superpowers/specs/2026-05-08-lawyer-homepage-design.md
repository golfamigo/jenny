# Jenny 案律師導向首頁改版設計

日期：2026-05-08

## 目的

將目前偏向「證據閱讀器」的首頁，改造成真正的「律師工作首頁」。

首頁的任務不再是直接展示證據卡，而是先讓律師與原告快速掌握：

- 程序目前走到哪裡
- 目前最重要的文書與其用途
- 核心爭點是什麼
- 律師現在需要決定哪些事項
- 原告現在還缺哪些證據或單據

證據、對話、書狀與訴之擴張頁面仍保留，但退到第二層入口，不在首頁直接主導閱讀順序。

## 頁面定位

- 頁面：`docs/index.html`
- 定位：案件總控台 / 律師工作首頁
- 主要讀者：律師、原告、協助整理案件的人
- 次要讀者：需要快速理解案件結構的第三人
- 閱讀順序：`程序 / 文書現況` → `核心爭點` → `待決策 / 待補件` → `第二層入口`

## 核心改動

### 1. 首頁主敘事改為程序與文書現況

首頁第一屏不再以證據卡為中心，而是先顯示案件總覽與程序資訊，例如：

- 案號：`114年度建字第109號`
- 程序階段：建字審理中
- 主路徑：修復、鑑定、訴之擴張
- 首頁定位：律師工作首頁

接著進入「程序 / 文書現況」區塊，條列目前最重要的程序節點：

- `114年度補字第632號民事裁定`
  - 訴訟標的價額 `523,898 元`
  - 補繳第一審裁判費 `5,730 元`
- `民事準備(一)暨聲請調查證據狀`
  - 已主張前手施工範圍
  - 已主張後手重作 / 保固責任
  - 已聲請鑑定漏水位置、原因、修復費用、修復期間與可居住性
- `目前程序焦點`
  - 共同被告結構
  - 補充鑑定題目
  - 訴之擴張項目與金額路徑

### 2. 首頁第二層才交代核心爭點

在程序 / 文書現況之後，用一個明確區塊整理本案四條核心責任線：

- 前手本體施工責任
- 後手重作、防水、保固責任
- 漏水因果需具體落到窗框、門檻、地坪與異材質交界
- 入住與付款不等於驗收完成或無瑕疵承認

這個區塊的目的，是讓律師不必先翻時間軸頁，也能在首頁看到目前訴訟的主軸。

### 3. Todo 明確拆成兩欄

首頁中段保留兩個待辦區塊，並明確區分責任：

#### 待律師討論 / 決策

這一欄放需要法律判斷與程序決定的事項，例如：

- 是否改列共同被告
- 訴之擴張主軸是否先鎖定修繕、鑑定、木地板、附帶裝修
- 遷居 / 住宿 / 搬運 / 倉儲先列事實還是先列金額
- 健康損害與精神慰撫金本次是否主張
- 鑑定事項補充到哪一層

#### 待原告補件 / 提供證據

這一欄放需要原告補齊的資料，例如：

- 最新修繕估價單
- 木地板拆除 / 重鋪 / 材料 / 工資單據
- 法院鑑定費與必要檢測費收據
- 住宿 / 搬運 / 倉儲 / 除霉除蟲清潔單據
- 醫療紀錄與診斷證明

### 4. 首頁不直接放證據縮圖

首頁不應再直接展開大量證據卡與圖片。

理由：

- 會破壞首頁作為「案件總控台」的角色
- 會讓讀者在還沒理解程序與爭點前就先掉進證據細節
- 會讓首頁與 `證據閱讀版` 的功能重疊

因此首頁只保留「重要入口卡」，不直接出現證據縮圖。

### 5. 證據頁降級為第二層入口

首頁底部保留文字型入口卡，供需要時再進：

- 時間軸 + 爭點
- 證據閱讀版
- 對話整理
- 書狀重點
- 訴之擴張

入口卡只說明用途，不直接展開內容。

## 不做的事

本次首頁改版不處理下列事項：

- 不重寫各子頁的內容本體
- 不改動證據閱讀版的證據卡細節
- 不在首頁新增新的法律分析長文
- 不在首頁直接加入完整訴之擴張矩陣
- 不把首頁做成待辦系統或資料庫式看板

## 與現有頁面的關係

- `index.html`：改成律師導向首頁
- 目前的證據閱讀結構保留，但不再作為首頁主體
- `timeline.html`、`conversation.html`、`pleading-summary.html`、`claim-expansion.html` 持續保留為第二層頁面

若需要，可考慮後續把原本首頁的「證據閱讀版」定位再明確標成子頁，但本次改版先不處理檔名重構。

## 驗證標準

改版完成後，首頁應同時滿足以下條件：

- 第一眼看到的是程序 / 文書現況，而不是證據牆
- 律師可以在首頁直接知道目前要先決定什麼
- 原告可以在首頁直接知道目前還缺哪些補件
- 證據頁入口仍清楚存在，但不搶首頁主敘事
- 首頁整體閱讀順序更接近「案件工作首頁」而不是「圖庫式證據頁」
